Deque（双端队列）的常见实现类

Deque（双端队列）接口在 Java 中有几个常见的实现类，每个都有其特定的用途和性能特点。Deque 的主要实现类有两个：**ArrayDeque** 和 **LinkedList**。此外，还有一个并发实现类 **ConcurrentLinkedDeque**。

1. ArrayDeque - 数组双端队列

这是 Deque 最常用、最通用的实现。

  • 内部数据结构：可动态扩容的数组。
  • 工作原理：它内部维护一个数组，并通过两个指针（head 和 tail）来标记队列的头和尾。当数组空间不足时，会进行扩容（通常容量翻倍）。
  • 性能特点：
    • 在两端添加、删除、获取元素的时间复杂度都是 **O(1)**，非常快。
    • 在中间插入或删除元素需要移动数组元素，时间复杂度为 **O(n)**。
  • 内存开销：较小。因为它只存储元素本身，没有额外的指针开销。
  • 线程安全：不安全。如果在多线程环境下使用，需要外部进行同步控制。
  • 适用场景：
    • 作为栈使用（push, pop）。
    • 作为普通队列使用（add, poll）。
    • 绝大多数单线程环境下的双端队列需求。

2. LinkedList - 链表双端队列

LinkedList 是一个“多面手”，它同时实现了 List 接口和 Deque 接口。

  • 内部数据结构：双向链表。每个节点都包含数据、指向前一个节点的指针和指向后一个节点的指针。
  • 工作原理：通过修改节点间的指针关系来添加或删除元素。
  • 性能特点：
    • 在两端添加、删除元素的时间复杂度是 **O(1)**。
    • 在中间插入或删除元素非常高效，也是 **O(1)**（前提是已经找到了要插入/删除的位置）。
    • 随机访问（通过索引 get(i)）很慢，需要从头或尾开始遍历，时间复杂度为 **O(n)**。
  • 内存开销：较大。每个元素都需要额外的空间存储前后节点的指针。
  • 线程安全：不安全。
  • 适用场景：
    • 当你需要频繁在列表中间进行插入和删除操作时。
    • 当你需要一个既支持 List 操作（按索引访问）又支持 Deque 操作的结构时（但要注意其随机访问性能差）。

3. ConcurrentLinkedDeque - 并发双端队列

这是 java.util.concurrent 包下的一个实现，专为高并发环境设计。

  • 内部数据结构：基于链表。
  • 工作原理：使用 CAS（Compare-And-Swap） 无锁算法来实现线程安全。
  • 性能特点：
    • 在高并发场景下，性能远优于使用 Collections.synchronizedDeque() 包装的队列。
    • 操作（如 add, poll）通常是非阻塞的，不会因为锁竞争而挂起线程。
    • 不能插入 null 元素。
    • **size() 方法的性能可能不是 O(1)**，因为它需要遍历链表来计算准确大小，在并发环境下这是一个耗时操作。应尽量避免调用。
  • 内存开销：较大（链表结构）。
  • 线程安全：安全。并且是高效的、无锁的线程安全。
  • 适用场景：
    • 高并发的生产者-消费者模型。
    • 任何需要多线程安全访问双端队列的场景。

总结与对比

特性 ArrayDeque LinkedList ConcurrentLinkedDeque
数据结构 数组 双向链表 链表 (CAS)
主要用途 栈、队列（首选） 频繁中间插入/删除 高并发场景
两端操作性能 O(1) (极快) O(1) (快) O(1) (并发下快)
中间插入/删除 O(n) (慢) O(1) (快) O(1) (并发下快)
随机访问 O(1) (快) O(n) (慢) O(n) (慢)
内存开销 小 大 大
线程安全 否 否 是 (无锁)

如何选择？

  • 默认选择：**ArrayDeque**。当你需要一个栈或队列时，它几乎总是最好的选择。
  • 需要频繁在中间操作：考虑 **LinkedList**。
  • 多线程高并发环境：必须使用 **ConcurrentLinkedDeque**。

另外，不要使用 Java 1.0 的 Stack 类，它已经被 ArrayDeque 完全取代。如果需要一个线程安全的栈，可以使用 Collections.synchronizedDeque(new ArrayDeque<>())。
